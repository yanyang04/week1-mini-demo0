# Week1 Mini Demo0 本地链文件存证智能合约Demo
## 1. 你做了什么
本周完成一套轻量级链上文件哈希存证最小Demo，完整走完智能合约开发实训流程：
1. 编写极简Solidity存证智能合约，实现文件哈希存储、重复哈希拦截、时间戳查询功能；
2. 基于Hardhat搭建本地区块链开发环境，编写合约部署脚本；
3. 启动Hardhat本地私有链，完成合约本地部署、写入测试哈希、读取校验全流程链交互；
4. 将合约源码、部署脚本、操作终端截图统一归档至GitHub仓库，形成可完整阅读的实训成果。

## 2. 哪部分是真实链上操作
全部操作在Hardhat本地私有链完成，本地节点具备完整区块链交易、区块、Gas机制，属于标准真实链环境操作：
1. 本地合约部署：启动本地私有链节点，发送部署交易，生成本地有效合约地址并写入区块；
2. 链上写入交互：调用合约`storeHash`方法，将自定义测试文件哈希存入本地链状态存储；
3. 链上读取校验：调用`getUploadTime`查询函数，从本地链读取已存哈希对应的区块时间，验证数据存储一致性；
4. 所有本地链操作终端截图统一存放在仓库`screenshot`文件夹。

## 3. 哪部分由AI辅助完成
1. 智能合约基础代码：AI生成mapping存储结构、基础读写函数、基础Solidity语法模板；
2. Hardhat部署脚本：AI生成基础deploy.js部署框架；
3. 文档结构：按照作业5项强制要求生成完整README写作框架；
4. 报错解决方案：针对合约编译报错、本地节点启动失败提供排查思路。

## 4. 你做了哪些人工判断和修改
1. 安全逻辑补充：AI原始代码没有重复哈希防重复存储逻辑，手动新增require判断，避免重复上链浪费Gas；
2. 版本语法修正：统一锁定Solidity 0.8.26稳定版本，利用内置溢出防护，删除过时语法；
3. 敏感文件隔离：手动配置.gitignore，过滤node_modules、.env私钥配置文件，防止隐私泄露；
4. 文档优化整理：删减AI冗余话术，严格按照作业5点要求分段排版，结构清晰方便阅读；
5. 流程梳理：整理本地启动、部署、交互完整操作步骤，方便他人复现Demo。
